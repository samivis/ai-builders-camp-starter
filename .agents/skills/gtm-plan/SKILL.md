# GTM Plan

Help the student write a 1-sentence pitch and a go-to-market plan, polish their README, and ship one real GTM action.

## When to use

When the student is on Day 7, or says "use the gtm-plan skill" / "help me with my pitch" / "how do I get people to use this."

## Replit rules
- For deployment: tell the student to use the Replit **Publish** button so the app has a live, shareable URL. There is no CLI deploy step.
- Keep the app simple: main logic in `main.py`, pages in `templates/`, styles in `static/`. Don't add new entry-point files or a build step.

## Workflow

### Step 1 — Find the user
Ask the student who the app is for — the one real person or group who has the problem it solves. Get specific.

### Step 2 — Write the pitch
Help them write a 1-sentence pitch anyone could understand in 5 seconds: what it is, who it's for, what it does. Iterate until it's clear and tight.

### Step 3 — Write the GTM plan
Create `gtm-plan.md` in the project root with:
- The 1-sentence pitch
- The target user
- 3 distribution channels that are safe for a teen builder: personal network, camp Discord, GitHub README, parent-approved social

### Step 4 — Polish the README
Update `README.md` so a stranger understands the app: what it does, who it's for, how to use it, and the live URL. Lead with the pitch.

### Step 5 — Ship one GTM action
Help the student do ONE real GTM action right now: post in the camp Discord, text a family member the link, or share with a friend. Pick something doable today.

### Done when
- [ ] `gtm-plan.md` exists with pitch, target user, and 3 safe channels
- [ ] `README.md` is polished and leads with the pitch
- [ ] One real GTM action has been shipped
